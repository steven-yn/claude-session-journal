# Journal Writing Skill

Claude Code 세션 데이터를 기반으로 회고 저널을 작성하는 스킬입니다.

## When This Skill Applies

- 사용자가 오늘/이번 주/이번 달 활동을 돌아보고 싶을 때
- 회고, 저널, 세션 요약을 요청할 때
- "오늘 뭐 했지?", "이번 주 정리해줘" 같은 요청

## How to Use

이 스킬이 활성화되면, 적절한 저널 커맨드를 안내하세요:

- `/journal-daily [YYYY-MM-DD]` — 일별 저널 생성
- `/journal-weekly [YYYY-WNN]` — 주별 저널 생성
- `/journal-monthly [YYYY-MM]` — 월별 저널 생성
- `/journal-status` — 저널 현황 조회
- `/journal-config` — 설정 조회/수정

## Writing Process (5단계 작성 프로세스)

저널 생성 시 반드시 아래 5단계를 순서대로 따르세요. 각 단계를 건너뛰지 마세요.
작업량이 많으면 병렬로 처리하세요.

### 1단계: 항목 분류 (Categorize)
세션 데이터를 분석하여 독립적인 항목으로 분류합니다.
- **별개의 주제**별로 분리 (프로젝트, 리서치, 글쓰기, 학습 등)
- **별개의 작업**별로 분리 (구현, 기획, 분석, 정리 등)
- **별개의 배움**별로 분리 (새로운 지식, 인사이트, 시행착오 등)
- 하나의 세션에 여러 주제가 섞여 있으면 각각 별도 항목으로 분리

### 2단계: 누락 검토 (Completeness Check)
분류한 항목 목록을 세션 데이터와 대조하여 빠진 것이 없는지 검토합니다.
- 모든 세션이 최소 하나의 항목에 반영되었는지 확인
- `files_modified` 목록에서 언급되지 않은 작업이 있는지 확인
- `user_queries`에서 놓친 작업 흐름이 있는지 확인
- 사소해 보여도 의미 있는 작업이면 포함

### 3단계: 내용 작성 (Flesh Out)
각 항목에 구체적인 내용을 채워 넣습니다.
- 세션 데이터의 `conversation_summary`, `user_queries`에서 맥락 추출
- "무엇을 했는가"보다 **"무엇을 배웠는가", "왜 그렇게 했는가"**에 집중
- 트러블슈팅은 **문제 → 접근 → 해결** 구조로 서술
- Raw 로그를 그대로 복사하지 말고 핵심 인사이트로 정제

### 4단계: 사실 검증 (Hallucination Check)
작성한 내용이 세션 데이터에 실제로 존재하는 사실에 기반하는지 검증합니다.
- **세션 데이터에 근거가 없는 내용은 삭제** — 추측으로 내용을 만들어내지 않음
- 구체적인 기술명, 라이브러리명, 파일 경로 등이 정확한지 확인
- "해결했다"고 쓴 문제가 실제로 해결되었는지 데이터에서 확인
- 통계 수치가 세션 데이터와 일치하는지 확인

### 5단계: 가독성 향상 (Polish)
최종 문서의 가독성과 품질을 높입니다.
- 자연스러운 문체로 다듬기 — 블로그 포스트로 재활용 가능한 수준
- 과도한 이모지나 장식 없이 깔끔하게
- 섹션 간 흐름이 자연스러운지 확인
- 코드 스니펫은 핵심적인 부분만, 설정의 `max_code_snippet_lines` 이내로
- 통계는 테이블 형태로 정리

## Writing Principles

### 핵심 인사이트 정제
- Raw 세션 로그를 그대로 복사하지 않음
- 의미 있는 배움, 결정, 시행착오만 추출
- "무엇을 했는가"보다 "무엇을 배웠는가"에 집중

### 문체와 톤
- 블로그 포스트로 재활용 가능한 자연스러운 문체
- 적당한 이모지나 장식 사용으로 깔끔하게
- 코드 스니펫은 핵심적인 부분만 포함

### 구조화
- 주제/맥락별로 활동 내용 그룹핑
- 어려움은 **상황 → 시도 → 결과** 구조로 기록
- 통계는 테이블 형태로 깔끔하게

### 다국어 지원
- 설정의 `resolved_language`에 따라 해당 언어로 전체 작성
- 섹션 제목도 해당 언어로 변경
